# PowerShot 開発ロードマップ

PowerShotを「IT現場で最強の証跡・エビデンス管理ツール」へと進化させるための計画です。

---

## 🚩 近期目標 (v3.x 系列) - 統合管理環境の構築

現在の「保存UI」をベースに、閲覧・編集・管理をすべて一箇所で行える「PowerShot Manager」を構築します。

### 1. PowerShot Manager (WPF版) の実装

- [ ] **統合プレビュー**: エクスプローラーでファイルを選択すると、メイン画面で即座に編集・再保存が可能に。
- [ ] **ビジュアル・エクスプローラー**: リスト表示にサムネイル（プレビュー画像）を統合し、ファイル名を見なくても直感的に探せるUIを実現。
- [ ] **HTML版の廃止**: すべての機能をWPFに集約し、ポータビリティをさらに向上。

### 2. 非破壊編集とデータ同期システムの導入

- [ ] **エビデンス保護（非破壊編集）**:
  - 編集前のオリジナル画像を `.powershot/originals/` へ自動バックアップ。
  - 編集ステート（アノテーションの座標やテキスト内容）を `index.json` で管理。
- [ ] **スマート同期機能**:
  - メインフォルダのファイル操作（削除・移動）を検知し、バックアップ側も自動追従。
  - ユーザーによる手動操作とManagerの状態を常に整合させる「セルフヒーリング」機能。

### 3. アノテーション機能（エディタの強化）

- [ ] **図形描画**: 矢印、四角枠、丸枠の追加。
- [ ] **秘匿情報保護**: モザイク（ぼかし）ツールの実装。
- [ ] **テキスト追加**: 画像内の任意の位置にテキストを配置。

### 3. UIアクセシビリティの改善

- [ ] **常駐トレイアイコン**: タスクトレイからのクイックアクセス。
- [ ] **モード切替の最適化**:
  - **Saverモード**: キャプチャ検知時に起動。新規画像を保護し、エクスプローラー選択時は別窓で開く。
  - **Managerモード**: 手動起動。同一ウィンドウ内でシームレスにプレビューを切り替え。

---

## 🚀 中期目標 (v4.0) - アーキテクチャの刷新とクロスプラットフォーム化

「Windows専用の便利ツール」から、プラットフォームを選ばない「プロフェッショナルツール」へ移行します。

### 1. Rust (Tauri) への移行

- **目的**: パフォーマンスの向上、バイナリサイズの軽量化、セキュリティ（署名問題）への対応準備。
- **UI**: WPFで培った統合デザインを、より軽量なフレームワークへ移植。

### 2. クロスプラットフォーム対応

- **macOS / Linux サポート**: Windows版と全く同じ操作感でエビデンス管理を可能に。

### 3. 配布・信頼性の向上

- **パッケージマネージャー対応**: Scoop, Homebrew 等への公式登録。
- **コードサイニングの検討**: 普及規模に応じた証明書の導入。

---

## 🌐 長期的な展望

- **プラグインシステム**: 自社の報告書フォーマットに合わせた自動リネーム機能などの拡張。
- **チーム共有設定**: チーム内でオーバーレイの形式や保存先ルールを共有する機能。
