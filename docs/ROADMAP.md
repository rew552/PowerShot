# SonicShot (formerly PowerShot) 開発ロードマップ

SonicShotは、スクリーンショット撮影からドキュメントへの貼り付けまでを「音速」で完結させるための、Rust製次世代エビデンス管理ツールです。

---

## 🚀 次世代フェーズ: SonicShot (v4.0) - Rustによる再構築

現在のPowerShell + WPFアーキテクチャから脱却し、Windowsに最適化された究極のパフォーマンスと配布の容易さを追求します。

### 1. アーキテクチャの刷新

- [ ] **Rustネイティブ化**: PowerShell/C#ランタイムを不要にし、単一の `.exe` で動作。
- [ ] **.NET依存の完全排除**: .NET 4.x/Desktop Runtimeのインストール状況に左右されない安定した動作。
- [ ] **起動の極限高速化**: PrintScreen押下からキャプチャ開始、UI表示までのレイテンシをミリ秒単位まで削減。

### 2. 「ドキュメント作成支援」への特化

- [ ] **スマート・クリップボード**:
  - 画像のコピーと同時に、**Markdown形式の画像リンク**（`![alt](path/to/img.png)`）を自動生成してクリップボードにセット。
  - エディタ（VS Code, Obsidian等）への貼り付けをシームレス化。
- [ ] **ディレクトリ相対パスの自動計算**:
  - プロジェクトルートを検知し、エディタからの相対パスでリンクを生成。
- [ ] **サイレント・キャプチャモード**:
  - UIを表示せず、背景で音速で保存・リンクコピーを行う「執筆特化」モード。

### 3. 配布と信頼性の向上

- [ ] **Microsoft Store への公開**:
  - ストア審査を通過させることで、ユーザーの信頼性を獲得し、SmartScreenの警告を回避。
- [ ] **単一バイナリ配布**:
  - アイコン、設定、GUIリソースをすべてexeに内包し、設定ファイル一つでポータブルに動作。

---

## 🚩 現行フェーズ (v3.x) - 安定化と移行準備

SonicShotへの完全移行までの間、現行のPowerShot環境のメンテナンスと移行データの整理を行います。

- [ ] **設定ファイルの共通化**: Rust版へ引き継げるように `settings.json` のスキーマを整理。
- [ ] **セッション管理の洗練**: 複数のドキュメント作成を並行して行える「セッション切り替え」の安定化。

---

## 🌐 長期的な展望

- **プラグインシステム**: 自社の報告書フォーマットに合わせた自動リネーム機能などの拡張。
- **モダンなアノテーション**: Rust/Tauri上での、GPU加速を利用した滑らかな図形描画・ぼかし機能。
- **クロスプラットフォーム（将来検討）**: Windowsでの完成後、macOS/Linuxへの移植性を考慮したコード設計。
