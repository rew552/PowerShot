# Release Notes

## v3.0 (v2.0 からの変更点)

PowerShot v3.0 は、v2.0 の「キャプチャ→保存」というシンプルなフローを保ったまま、**画像編集・設定永続化・独立型ビューア** を追加した大型アップデートです。

---

### ✨ 新機能

#### 画像編集機能 (新設)

- **トリミング**: プレビュー上でドラッグして切り抜き範囲を指定できます。エッジ・コーナーをドラッグしてサイズ変更、内部をドラッグして移動も可能。X/Y/W/H を直接数値入力することも可能です。
- **オーバーレイ (テキスト合成)**: 画像の四隅 (左上 / 右上 / 左下 / 右下) のいずれにも、任意のテキストとシステム情報を合成できるようになりました。v2.0 では「左下に固定」だったシステム情報も、自由に配置できます。

#### ⚙️ 設定画面 (新設)

- 画面右上の歯車アイコンから設定ダイアログを開けます。
- 保存先フォルダの変更 (フォルダ選択ダイアログ対応)
- JPEG 品質スライダー
- タイムスタンプフォーマットのテンプレート選択 / カスタム
- 設定は `src/settings.json` に自動保存され、次回起動時に復元されます。

#### 🖼️ 独立型 画像ビューア (新設)

- 新しい `powershot-viewer.bat` で、ブラウザ内で動作する軽量な画像ビューアを起動できます。
- フォルダツリー、ファイル名検索、ソート (名前順 / 日時順)
- サムネイル一覧 → クリックで拡大、`←` / `→` キーで前後の画像へ。
- 完全にローカルファイルのみで動作 (外部依存なし)。

---

### ⚡ パフォーマンス・安定性

- **イベント駆動なクリップボード監視**: v2.0 の定期ポーリング方式から、`WM_CLIPBOARDUPDATE` メッセージフックに移行。アイドル時の CPU 使用がほぼゼロになりました。
- **重複画像の検出**: SHA256 ハッシュで直前の画像と同一ならポップアップしないようにし、誤発火を抑制。
- **Excel セルコピーの除外**: クリップボードに `XML Spreadsheet` / `HTML Format` が含まれる場合 (= Excel セルコピー) はキャプチャを発動しません。
- **小画像フィルタ**: 20×20 未満の画像はスクリーンショット候補から除外。
- **DPI 対応**: `SetProcessDPIAware` を呼び出し、高 DPI ディスプレイでの座標ズレを解消。
- **連続押下時のプレビュー再描画をデバウンス**: クロップ用の数値入力時、150ms 以内の連打は 1 回の再描画にまとめます (4K 画像のフルクローン削減)。
- **システム情報のキャッシュ化**: ホスト名・IP の DNS 解決を 1 セッション 1 回に。

---

### 🔒 安全性

- **ルート境界チェック**: エクスプローラーのパス遷移は保存先フォルダ以下に制限され、上位への遡りを禁止。
- **削除確認ダイアログ**: ファイル / フォルダ削除時には常に確認ダイアログを表示。
- **ファイル名禁則文字の検証**: 保存・フォルダ作成時に `\ / : * ? " < > |` 等を事前チェック。

---

### 🏗️ 内部アーキテクチャ

v2.0 は単一の `.ps1` + `.cs` 構成でしたが、v3.0 では責務ごとに分割されたモジュラー構造になりました:

```
src/
├── App/                   … エントリポイント・クリップボード監視
├── Controllers/           … MainWindow / Settings / Viewer / Crop の各コントローラ
├── Models/                … AppSettings / SessionState / ExplorerItem 等
├── Utils/                 … FileManager / IconHelper / OverlayRenderer / SequenceManager
└── Views/                 … MainWindow / Settings / Preview / Viewer の XAML / HTML
```

- **OverlayRenderer**: オーバーレイ描画とシステム情報取得を純粋関数として独立化。
- **CropController**: トリミングのドラッグ判定 / カーソル制御 / テキストボックス同期を独立クラスに。
- **SettingsManager**: `DataContractJsonSerializer` で `settings.json` をシリアライズ。
- **IconHelper**: `SHGetFileInfo` で取得した拡張子別アイコンを LRU キャッシュ。

### 📦 互換性

- v2.0 で利用していた `Screenshots` フォルダはそのまま使用できます。
- v2.0 には設定ファイルが存在しなかったため、初回 v3.0 起動時に `src/settings.json` がデフォルト値で自動生成されます。
