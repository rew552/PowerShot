# PowerShot v2.0

スクリーンショットのファイル名を自動生成し、指定したフォルダへ保存するための支援ツールです。

## 主な機能

- **クリップボード監視**: スクリーンショット（または画像）をコピーすると、自動で保存用UIが立ち上がります。
- **ファイル名全自動生成**: 接頭辞・オプション・連番を自在に組み合わせてファイル名を生成します。
- **連番の自動インクリメント**: 保存フォルダ内の既存ファイルをスキャンし、次の番号を自動的に割り当てます。
- **桁数カスタマイズ**: 連番の桁数を1〜6桁の間で自由に設定・変更できます。
- **簡易エクスプローラー**: 保存先のフォルダ構成をUI上で確認・移動でき、連番の計算も現在の移動先に同期します。
- **フォルダ作成・削除**: UI上から直接新しいフォルダを作成したり、不要なファイルを削除したりできます。
- **ダークテーマ対応**: 作業を邪魔しない落ち着いたダークテーマUI。

## 使い方

1. `powershot.bat` を実行します（バックグラウンドでクリップボードの監視が始まります）。
2. 通常通りスクリーンショット（`Win + Shift + S` など）を撮ります。
3. 自動で PowerShot のウィンドウが表示されるので、必要に応じて「接頭辞」や「オプション」を入力または変更します。
4. `Enter` キー または `保存` ボタンで、指定したフォルダに画像ファイルとして保存されます。
   - `Esc` キーで保存せずに閉じることができます。
5. ターミナルまたはバッチ実行ウィンドウで `Ctrl + C` を入力するとアプリを終了できます。

## フォルダ構成

- `powershot.bat`: アプリ起動用バッチファイル。
- `src/`: アプリケーションのプログラム本体（PowerShell, C#, XAML）。
- `Screenshots/`: デフォルトの画像保存先フォルダ。
- `readme.md`: このファイル。

## 注意事項

- 本ツールは PowerShell 5.1 以降および .NET Framework 4.6 以降が必要です。
- クリップボードに表データ（Excelなど）がコピーされた場合は、誤作動防止のためUIを表示しない仕様になっています。
